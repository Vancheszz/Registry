# Ручное тестирование через Selenium (Chrome)

Данные сценарии помогают вручную проверить веб-интерфейс: вход и создание записи на приём.
Файлы расположены в каталоге `tests/ручное тестирование/`.

## Файлы
- `selenium_login_and_booking.py` — автоматизирует шаги входа и создания приёма через Chrome WebDriver.
- `selenium_manual.md` — это описание (вы читаете его сейчас).

## Подготовка среды
1. Установите зависимости: `pip install -r backend/requirements.txt` (включает selenium).
2. Скачайте совместимый **ChromeDriver** и убедитесь, что его путь прописан в переменной окружения `CHROMEDRIVER` или добавлен в `PATH`.
3. Запустите backend (`uvicorn main:app --reload --port 8000` внутри каталога `backend`) и frontend (например, `npm start` для директории `frontend`), чтобы UI был доступен.
4. Проверьте наличие тестового пользователя с логином/паролем, указанными в переменных окружения или дефолтных значениях скрипта.

## Как запускать
Запуск автоматизированного сценария (используется Chrome):
```bash
CHROMEDRIVER=/path/to/chromedriver \
SELENIUM_BASE_URL=http://localhost:3000 \
SELENIUM_USERNAME=doctor \
SELENIUM_PASSWORD=doctorpass \
python "tests/ручное тестирование/selenium_login_and_booking.py"
```

Скрипт откроет браузер, выполнит вход и создаст запись на приём через форму интерфейса. После выполнения окно браузера останется открытым на несколько секунд для визуальной проверки.

## Что проверяем вручную
1. **Вход:** открывается страница авторизации, вводятся логин/пароль, происходит переход на дашборд.
2. **Создание записи:** в расписании появляется новая запись с выбранной датой и временем; после сохранения она отображается в списке.

## Подсказки
- Если элементы интерфейса меняли селекторы, обновите их в `selenium_login_and_booking.py`.
- Для повторного запуска удобно сохранять базовые URL и креды в `.env` и экспортировать их перед стартом.
